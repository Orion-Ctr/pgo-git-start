## Goal:
• understand the relationship between the working directory, the staging area and the repository 
history 
• use git init, git status, git add, git commit and git log in practice 
• create a public GitHub repository and perform the first push 
• practice working on feature branches and merging them into main 
• optionally: create and resolve a merge conflict manually

## Commands Used
*1 `git init`: Initializes the repository.
*2 `git add`: Adds files to the staging area.
*3 `git commit`: Saves a snapshot of the changes.
*4 `git checkout -b`: Creates a new branch and switches to it.
